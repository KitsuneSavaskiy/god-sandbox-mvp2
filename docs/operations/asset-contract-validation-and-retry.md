# Asset Contract Validation and Retry Plan

Sprint 10-E — Non-empty content checks added. Event-standing-expressions is a first-class validation lane.

## Overview

The asset contract validator (`validate-asset-contract.mjs`) is a local quality gate
that checks generated PNG assets against machine-readable contracts.

It runs **three gate types** in sequence (see below).
Semantic quality (pose consistency, character identity, row order) still requires human review.

## Gate types

| Gate | When it runs | Can fail? |
|---|---|---|
| **Structural** | Always | Yes — file-exists, dimensions, alpha |
| **Pixel-margin** | When file decodes as 8-bit RGBA | Yes — visible pixels inside safe margin zone |
| **Required-content** | Expression sets — when file decodes as 8-bit RGBA | Yes — fully transparent file has no visible pixels |
| **Human review** | Portrait expression sets | Flags `identityConsistencyNeedsHumanReview: true` |

`marginCheckStatus` in the report reflects the pixel gate:
- `"pass"` — all frames/images checked, no safe-margin violations
- `"fail"` — at least one frame has a safe-margin violation (`pixel-margin` check added to `checks[]`)
- `"not-run"` — PNG could not be decoded (header-only stub, non-RGBA, or decode error)

## What the validator checks

| Check | Gate | Description |
|---|---|---|
| `file-exists` | structural | Required asset file is present at the expected path |
| `png-signature` | structural | First 8 bytes match the PNG magic number |
| `dimensions` | structural | Canvas width and height match the contract exactly |
| `alpha-channel` | structural | Color type includes alpha (type 4 = grayscale+alpha, type 6 = RGBA) |
| `canvas-size-consistency` | structural | All expression files in a set share the same canvas size |
| `pixel-margin` | pixel | No visible pixel (alpha > 0) falls inside the safe margin zone of its frame |
| `required-content` | pixel | At least one non-transparent pixel exists in the image (expression sets only) |

## What the validator does NOT check

- Semantic row quality (correct motion per row, consistent pose across frames)
- Character identity (does the sprite depict the intended character?)
- Pose sameness across expression variants
- Whether the asset was generated by the approved pipeline

**identityConsistencyNeedsHumanReview** is set to `true` in the report for portrait
expression sets because semantic verification of character identity across expressions
cannot be automated.

## How to run

```bash
# Validate a combined PO preview sheet
npm run assetgen:validate-contract -- \
  --slug ryo \
  --contract resident-po-combined-preview-v1 \
  --asset-dir assets/generated/residents/ryo/incoming

# Validate canonical two-sheet sprites
npm run assetgen:validate-contract -- \
  --slug ryo \
  --contract resident-canonical-two-sheet-v1 \
  --asset-dir assets/generated/residents/ryo/incoming

# Validate portrait expressions
npm run assetgen:validate-contract -- \
  --slug ryo \
  --contract portrait-expression-set-v1 \
  --asset-dir assets/generated/residents/ryo/incoming

# Validate event standing expressions
npm run assetgen:validate-contract -- \
  --slug ryo \
  --contract event-standing-expression-set-v1 \
  --asset-dir assets/generated/residents/ryo/incoming

# Dry-run: print what would be validated, do not write files
npm run assetgen:validate-contract -- \
  --slug ryo \
  --contract resident-po-combined-preview-v1 \
  --asset-dir assets/generated/residents/ryo/incoming \
  --dry-run

# Help
npm run assetgen:validate-contract -- --help
```

## Contract IDs

| Contract ID | Expected files | Canvas size |
|---|---|---|
| `resident-canonical-two-sheet-v1` | `resident-sprite-sheet.png`, `resident-sprite-sheet-extended.png` | 1536×1872 each |
| `resident-po-combined-preview-v1` | `resident-sprite-sheet-combined.png` | 826×1904 |
| `portrait-expression-set-v1` | `expressions/neutral.png`, `happy.png`, `angry.png`, `sad.png`, `surprised.png` | Per-file (consistent) |
| `event-standing-expression-set-v1` | `event-expressions/neutral.png`, `happy.png`, `angry.png`, `sad.png`, `surprised.png`, `worried.png`, `determined.png`, `shocked.png` | Must be consistent across all files |

## Output files

Reports are written to `assets/generated/residents/<slug>/qa/` (gitignored).

### asset-contract-report.json

```json
{
  "candidateOnly": true,
  "slug": "ryo",
  "contractId": "resident-po-combined-preview-v1",
  "generatedAt": "2026-05-25T00:00:00.000Z",
  "passCount": 4,
  "failCount": 1,
  "hardGatePassed": false,
  "qualityGateStatus": "fail",
  "marginCheckStatus": "fail",
  "contentCheckStatus": "not-run",
  "checks": [
    { "check": "file-exists", "path": "...", "passed": true },
    { "check": "png-signature", "path": "...", "passed": true },
    { "check": "dimensions", "path": "...", "passed": true, "expected": "826x1904", "actual": "826x1904" },
    { "check": "alpha-channel", "path": "...", "passed": true },
    { "check": "pixel-margin", "passed": false, "side": "left", "actual": 5, "required": 8,
      "row": 0, "column": 2, "reason": "row 0 col 2: left margin 5px < 8px required",
      "scope": "frame-only" }
  ]
}
```

`marginCheckStatus` values:
- `"pass"` — all frames scanned, no violations
- `"fail"` — at least one `pixel-margin` check in `checks[]`
- `"not-run"` — PNG could not be fully decoded (header-only file, non-RGBA, or decode error); or contract type does not run margin checks

`contentCheckStatus` values (expression sets only):
- `"pass"` — all decoded expression files have at least one visible pixel
- `"fail"` — at least one `required-content` check in `checks[]` (fully transparent image)
- `"not-run"` — PNG could not be fully decoded, or contract type does not run content checks (sprite sheets)

For portrait expression sets, an additional field is included:
```json
  "identityConsistencyNeedsHumanReview": true
```

Human-readable version of `retry-plan.json`, formatted as markdown with

## Retry plan scope values

Each failure in `retry-plan.json` has a `scope` field indicating what needs to be regenerated:

| Scope | Meaning | Typical check type |
|---|---|---|
| `full-sheet` | The entire sprite sheet must be regenerated | `dimensions`, `alpha-channel`, `png-signature` |
| `row-only` | Only specific rows need to be regenerated | (reserved) |
| `frame-only` | A specific frame violates the safe margin — character content too close to cell edge | `pixel-margin` |
| `expression-only` | Portrait expression file(s) need to be regenerated | `file-exists`, `alpha-channel`, `pixel-margin`, `required-content` |
| `event-expression-only` | Event standing expression file(s) need to be regenerated | `file-exists`, `alpha-channel`, `pixel-margin`, `required-content` |

## identityConsistencyNeedsHumanReview

This field is set to `true` in the report when the contract covers portrait
expression sets (`portrait-expression-set-v1`).

It signals that the automated validator has confirmed structural requirements
(files exist, valid PNG, alpha channel present) but **cannot verify**:

- Whether all expression files depict the same character
- Whether expressions match the reference portrait
- Whether expressions are visually plausible for the character's personality

A human reviewer must confirm character identity and expression quality before
promoting any expression set.

## Security

The validator refuses to write output to `public/art/**`. If `--asset-dir` or the
derived output path resolves under `public/art/`, the tool exits with a SECURITY error.

This prevents accidental promotion of candidate assets to production art paths.

## Exit codes

| Code | Meaning |
|---|---|
| 0 | Validator ran successfully (check `failCount` and `marginCheckStatus` in the report) |
| 1 | Fatal error (bad arguments, unknown contract, security violation) |
| 2 | Validation failures found — only emitted when `--fail-on-violation` flag is set |

Exit code 0 does not mean all checks passed. Read `failCount` and `marginCheckStatus`
in `asset-contract-report.json` for the full result.

Use `--fail-on-violation` in CI pipelines when you want a non-zero exit code on any
structural or pixel-margin failure.
