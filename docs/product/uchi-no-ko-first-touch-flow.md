# うちの子 初回候補づくりフロー

状態: 補助ドキュメント

## 目的

キャラクター素材のローカル候補づくりでは、ユーザーに「生成管理」ではなく「候補づくりの制作メモ」として進行を見せる。

ユーザーが `.godsandbox` 配下のファイルを読まなくても、次の4点を分かるようにする。

- いま何が進んでいるか
- 何を待っているか
- どこで止まったか
- 次に何をすればよいか

## 表示方針

主表示では、生の jobId、内部 status、ファイルパスを前面に出さない。

代わりに、次のような日本語で見せる。

| 内部状態 | 主表示 |
|---|---|
| `pending` | 制作依頼を受け取りました |
| `prompt-pack-ready` | この子の特徴をまとめています |
| `gen2-dispatched` | ローカル補助に渡しました |
| `watcher-intake-done` | 制作の準備が整いました |
| `error` | うまく進めませんでした |

開発者が必要とする jobId、status、作業ファイル、確認レポート、修正案ファイルは、折りたたみの「開発者向けの詳細」に置く。

## 失敗時の案内

自動確認の失敗は、ユーザーが直す画像を判断できる言葉に変換する。

| check | 主表示 |
|---|---|
| `file-exists` | 必要な候補がまだ届いていません |
| `required-content` | 画像が透明だけです |
| `alpha-channel` | 背景が透過されていません |
| `dimensions` | 画像サイズが規格と違います |
| `pixel-margin` | 端に寄りすぎています |
| `canvas-size-consistency` | 表情画像のサイズが揃っていません |

元の check 名、reason、expected、actual は隠さず、詳細欄に残す。

## 確認パック

確認パックは、候補画像を見比べるためのローカルHTMLである。

ブラウザから直接開くAPIがない場合でも、UIには次を表示する。

```bash
npm run assetgen:review-pack -- --slug <slug>
```

生成先:

```text
assets/generated/residents/<slug>/review-pack/index.html
```

## イベント用の表情

イベント用の立ち絵表情を依頼した場合、UIは必要な8種類を隠さず表示する。

- 通常
- 笑顔
- 怒り
- 悲しみ
- 驚き
- 心配
- 決意
- 衝撃

## 安全境界

このフローは候補づくりだけを扱う。

- 外部APIは呼ばない
- `public/art` へ書き込まない
- ready 昇格をしない
- テスト用の仮連携は、PO確認に使えないことを明示する
