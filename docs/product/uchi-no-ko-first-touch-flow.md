# うちの子を箱庭に迎える初回フロー

## 目的

非技術者が、1枚絵・名前・ひとこと性格だけで新しい子の制作候補を準備できるようにする。

この画面では候補を作るだけで、箱庭の正式採用は自動では行わない。候補を見て、この子らしいか確認してから使う。

## 画面の言葉

- 画面名: この子を箱庭に迎える
- 主ボタン: 箱庭に迎える
- 候補確認: 候補を見る
- 内容確認: この子らしさを確認する

内部向けの lane / bridge / manifest / contract / job queue は、通常表示では見せない。必要な人だけ `技術詳細` を開いて確認する。

## 入力

- 見た目画像
- 名前
- ひとこと性格

任意入力:

- 口調
- 年齢
- 技術詳細

口調が空欄の場合は「その子らしい自然な話し方」として扱う。年齢が空欄の場合は内部処理用に 18 を使う。

## プリセット

| 表示名 | 作る候補 | イベント用の立ち姿表情 |
| --- | --- | --- |
| まず箱庭に来てもらう | 歩く姿、顔まわりの表情、小さな絵 | 含めない |
| 表情を増やす | 顔まわりの表情、イベント用の表情 | 含める |
| イベントでも使えるようにする | 歩く姿、顔まわりの表情、イベント用の表情、小さな絵 | 含める |

## ローカル補助が止まっているとき

生の通信エラーは見せない。

表示する文:

> ローカル補助がまだ準備中です

あわせて、実行コマンドとして `npm run assetgen:server` を見せる。

## 技術詳細

次の項目は折りたたみに置く。

- jobId
- portraitPath
- lanes
- bridge mode
- previewMode

fake bridge の場合は、通常表示でも次の注意を出す。

> これは動作確認用です。見た目候補としては使えません。
